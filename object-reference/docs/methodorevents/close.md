<h1 class="heading"><span class="name">Close</span> <span class="right">Event 33</span></h1>

[**Applies To**](../methodoreventapplies/close.md)

**Description**


A Close event is generated when an object is destroyed.



For a [Form](../objects/form.md) or [SubForm](../objects/subform.md), the event may be generated by the user selecting "Close" from its System Menu. In this case, the event is reported **before** the window is destroyed, and you may prevent it from going ahead by associating a callback function which returns a result of 0.



By trapping this event you can control termination of your application in many different ways. For example, you could:

- automatically close all [Forms](../objects/form.md) in your application when the master [Form](../objects/form.md) is closed.
- prevent the user from terminating the application if it is inappropriate at that time.
- display an "Are you sure ?" [MsgBox](../objects/msgbox.md).



Note that although all the object's child GUI objects will also be destroyed, none of the child objects will generate a Close event.
